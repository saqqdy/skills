# MCP、Agent、Skills 概念指南

## 概念解释

### MCP (Model Context Protocol)
Anthropic 推出的**开放协议**，让 AI 助手能连接外部工具和数据源。比如：
- 连接数据库、文件系统、API
- 让 AI 能"看到"和"操作"你的实际环境
- 类似于"AI 的 USB 接口"

### Agent（代理）
AI 执行复杂任务的**自主实体**。它能：
- 拆解任务、多步骤执行
- 自主决策、调用工具
- 比如我现在就是一个 Agent，能帮你写代码、跑命令、操作文件

### Skills（技能）
**预定义的能力模块**，扩展 AI 在特定领域的专业性。比如：
- `vue` - Vue 开发最佳实践
- `vitest` - 测试框架指导
- `pnpm` - 包管理器使用

## 作为前端/Node 开发者你能做什么

### 1. 日常开发提效
```
"帮我写一个 Vue 组件"
"运行测试并修复失败的用例"
"重构这个函数，让它更易读"
```

### 2. 代码探索和理解
```
"这个项目的架构是什么？"
"这个函数在哪里被调用？"
```

### 3. 自动化任务
```
"创建一个定时任务，每 5 分钟检查构建状态"
"帮我提交代码，写个好用的 commit message"
```

### 4. MCP 扩展（进阶）
- 安装 MCP server 让 AI 连接你的工具（如 GitHub、数据库、Jira）
- 自己写 MCP server 让 AI 操作你的内部系统

## 如何入手

**最简单的方式：直接用**

现在就可以让 AI 帮你做事，比如：
- "帮我看看这个项目结构"
- "写个工具函数处理 xxx"
- "运行 npm test"

**查看可用技能：**
```
/find-skills
```

**配置 MCP server：**
```bash
# 安装一个 MCP server，比如 GitHub
claude mcp add github
```

## 推荐学习路径

1. **入门**：直接使用 Claude Code 进行日常开发
2. **进阶**：熟悉各种 Skills，提高特定领域效率
3. **高级**：配置 MCP server，连接你的工具链
4. **专家**：自己开发 Skills 和 MCP server
